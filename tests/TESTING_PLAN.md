# План тестирования

## 1. Цели и задачи тестирования

Обеспечить корректность работы функций вычисления площади и периметра для круга и квадрата. Выявить возможные ошибки при граничных и типичных входных значениях.

## 2. Описание тестируемого продукта

Проект `geometric_lib` представляет собой библиотеку на языке Python, предназначенную для вычисления геометрических характеристик двух фигур: круга и квадрата.  
Содержит следующие функции:
- `circle.area(r)` — вычисление площади круга по радиусу;
- `circle.perimeter(r)` — вычисление длины окружности;
- `square.area(a)` — вычисление площади квадрата по длине стороны;
- `square.perimeter(a)` — вычисление периметра квадрата.  

Библиотека не имеет сторонних зависимостей и использует только стандартную библиотеку Python (`math`).

## 3. Область тестирования

Тестированию подлежат все четыре функции проекта:
- `circle.area`
- `circle.perimeter`
- `square.area`
- `square.perimeter`

Проверяется корректность возвращаемых значений при различных входных данных.

## 4. Стратегия тестирования

Применяется **функциональное white-box тестирование** с использованием фреймворка `unittest`.  
Тест-кейсы охватывают:
- **нулевые значения** входных параметров (например, радиус = 0, сторона = 0);
- **положительные целые и дробные числа**;
- **большие значения** (для проверки устойчивости к переполнению и ошибкам округления).

Для функций, использующих число π (`math.pi`), применяется `assertAlmostEqual` с допустимой погрешностью `1e-10`.

## 5. Критерии приемки

- Все тесты проходят успешно (статус *OK* при запуске через `unittest`);
- Покрыты как типичные, так и граничные случаи использования;
- Не обнаружено логических ошибок или несоответствий спецификации;
- Тесты легко запускаются и воспроизводимы на любой системе с Python 3.x.

## 6. Ожидаемые результаты

- Все тесты завершаются успешно без ошибок и сбоев;
- Подтверждена корректность реализации всех функций;
- Сформирован набор автоматизированных unit-тестов, пригодный для дальнейшего использования при модификации кода;
- Тесты размещены в публичном репозитории и отражены в истории проекта.
